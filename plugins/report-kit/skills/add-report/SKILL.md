reports/ に新しい調査タスクファイルを作成します。

## 引数

$ARGUMENTS

引数としてタスクのファイル名タイトル（英語・ハイフン区切り）を指定できます（例: `/add-report search-api-limits`）。
引数がない場合は「知りたいこと」から自動生成します。

## 手順

### 1. コンテンツの収集

ユーザーが同一メッセージ内にすでに「知りたいこと」や「目的」を記載している場合はそれを使用する。

記載がない場合は以下を確認する:

- **知りたいこと**（必須）: 具体的に何を調べたいか
- **目的**（任意）: なぜこの調査が必要か。省略可能である旨を伝える

### 2. タイトルの確定

タイトルは **ファイル名用（英語）** と **見出し用（日本語）** の2種類を用意する。

- `$ARGUMENTS` が指定されている場合: それをファイル名タイトルとして使用する。日本語見出しは「知りたいこと」の内容を要約して生成する
- `$ARGUMENTS` がない場合: 「知りたいこと」の内容から Claude が適切なファイル名（英語・ハイフン区切り）と日本語見出しを生成する

例:
- 知りたいこと: 「Enterprise Search の検索 API のレート制限」→ ファイル名 `search-api-rate-limit`、見出し `Enterprise Search 検索 API レート制限`

### 3. 連番の計算

Bash で `ls reports/ 2>/dev/null | grep -E '^[0-9]{4}_' | sort | tail -1` を実行し、`reports/` 配下にあるタスクディレクトリから最大番号を取得する。出力されたディレクトリ名の先頭4桁を最大番号として読み取り、+1 を4桁0パディングで次の番号とする（例: 出力が `0003_foo` なら次は `0004`）。出力が空の場合は `0001` を使用する。

### 4. ファイルの生成

Write ツールで `reports/{padded}_{ファイル名タイトル（英語）}/{padded}_{ファイル名タイトル（英語）}.md` を作成する（Write は親ディレクトリを自動作成する）。

ファイルの内容:

```
# {日本語タイトル}
## 知りたいこと

{知りたいこと の内容}

## 目的

{目的 の内容（省略時は空行のまま）}
```

### 5. report タスクの実行

作成したファイルのパスをユーザーに伝えた後、AskUserQuestion で「続けて `/report` を実行しますか？」を確認する。選択肢は「はい（すぐ実行する）」と「いいえ（後で実行する）」を用意する。

ユーザーが「はい」を選んだ場合、Skill ツールで `report` スキルを起動する。このとき **手順3で採番した実際の4桁番号を `args` パラメータに文字列として必ず渡す**こと（例: 採番が `0004` なら `args: "0004"`）。`{xxxx}` はプレースホルダーであり、実際の番号に置き換えること。`args` を省略したりプレースホルダーのまま渡すと、フォールバックロジックにより別のタスクが実行される危険がある。

## 注意事項

- ファイル名は **ハイフン区切り**（例: `search-api-rate-limit`）
- ファイル名の英語タイトルと本文の日本語タイトルは別々に管理する
- **git commit は行わない**
