# Report ワークフロー詳細手順書

## ログ記録の原則

ログはインタラクティブセッションで得た情報・判断・経緯の**完全な記録**である。report ファイルの `## 調査サマリー` は要約版、ログは完全版。**情報の要約・省略・圧縮をしてはならない。**

- 調べたファイルごとに発見した事実を具体的に書く（結論だけでなく内容そのものを記録する）
- 会話・やり取りも省略せず時系列で記録する

## 命名規則

- ディレクトリ名: `{xxxx}_{title}/`
- タスクファイル名: `{xxxx}_{title}.md`（ディレクトリ名と同じ basename）
- ログファイル名: `log.md`（固定）
- `xxxx`: 4桁の0パディング連番（例: `0001`, `0002`）
- `title`: 作業タイトル（スペースなし、ハイフン区切り推奨）

## タイムスタンプ

JSTタイムゾーンの ISO 8601 形式を使用する。

```bash
TZ=Asia/Tokyo date +"%Y-%m-%dT%H:%M:%S+09:00"
```

出力例: `2026-04-11T21:30:00+09:00`

## report ファイルの基本構造

report ファイルはユーザーが以下の構造で作成する。`## 目的` セクション（Why）は推奨項目だが必須ではない。省略した場合は `/report` コマンド実行時にユーザーへの確認を経て続行できる。

```markdown
# タイトル
## 知りたいこと
（具体的に何を調査してほしいか）

## 目的
（なぜこの調査が必要か — 背景・動機・ゴール）※ 推奨、省略可
```

- `## 目的`: セクション名は固定しないが、調査の動機・背景・理由（Why）を記載することを推奨する。省略する場合は `/report` 実行時に確認が入る
- `## 知りたいこと`: 具体的な調査内容（What）を記載する

## report ファイルへの追記テンプレート

report ファイルへの追記は以下の構造で行う。タスク内容はユーザーが記述し、以降のセクションを Claude が追記する。

```markdown
## 調査サマリー

（調査で判明した重要事項の要約。詳細はログファイルを参照。）

- 判明した主要事実の箇条書き
- 引用した根拠ドキュメント
- 未解決の疑問点（あれば）

---

## 完了サマリー

- **完了日時**: 2026-04-11T21:30:00+09:00
- **ログファイル**: `reports/{xxxx}_{title}/log.md`
- **備考**: （任意）
```

## ログファイルテンプレート

`reports/{xxxx}_{title}/log.md` に以下の構造で記録する。

```markdown
# {xxxx}_{title} 調査ログ

## 基本情報

- **タスクファイル**: reports/{xxxx}_{title}/{xxxx}_{title}.md
- **開始日時**: 2026-04-11T21:00:00+09:00
- **完了日時**: 2026-04-11T21:30:00+09:00

## タスク概要

（report ファイルの要望セクションの内容を転記する）

## 調査ファイル一覧

（調査したドキュメントのパス一覧。調査の網羅性確認用。）

- `docs/example/foo.md`
- `docs/reference/bar.md`

## 調査結果

（調査した内容を**省略せず詳細に**記述する。
調べたファイルごとに、発見した具体的な事実・構造・記述内容を書くこと。
「〜を確認した」のような結論だけでなく、確認した内容そのものを書く。
引用・具体例・行番号を含める。要約禁止。）

### `docs/example/file.md`

（このファイルで発見した事実を詳細に記述）

## 調査アプローチ

（調査の進め方、検索キーワード、調査の経緯を記録する。
どのような仮説を立て、どのように検索・調査したかを書く。）

## 会話内容

（ユーザーの指示とClaudeの応答を時系列で記録）

### [21:00] ユーザー指示
（指示内容）

### [21:05] Claude 対応
（応答・調査内容）

## 判断・意思決定

- （調査上の判断・解釈のトレードオフの記録）

## 問題・疑問点

- （調査中に発生した問題や未解決の疑問）
```

## 段階的ログ記録ルール

**重要**: ログは作業完了後にまとめて書くのではなく、段階的に追記すること。

1. **調査開始時（最初のステップ）**: ログファイルを作成し、基本情報とタスク概要を記入する
   - ヘッダー、基本情報（開始時刻）を書く（完了日時は TBD）
   - 「タスク概要」に report ファイルの要望を転記する
   - 「調査ファイル一覧」と「調査結果」は調査を進めながら随時追記する

2. **調査中（各ファイル・ドキュメントを調べるたびに）**: ログファイルへ追記する
   - 「調査ファイル一覧」に調査したファイルを追記する
   - 「調査結果」に、そのファイルで発見した内容を**省略せず詳細に**追記する（要約禁止）
   - 重要な判断をしたら「判断・意思決定」セクションに追記する
   - 問題・疑問が生じたら「問題・疑問点」セクションに追記する

3. **調査完了時**: 最終化する
   - ログファイルの完了日時を更新する
   - 「調査アプローチ」セクションに調査の進め方・経緯を記述する
   - 「会話内容」セクションにユーザーとのやり取りを記述する（省略せず）
   - report ファイルへ `## 調査サマリー` と `## 完了サマリー` を追記する

## タスク検出ロジック

- 未完了タスク: `reports/{xxxx}_{title}/{xxxx}_{title}.md` で `## 完了サマリー` を含まないもの
- `/report` コマンドに引数がない場合: 未完了タスクのうち番号が最大のものを選択
