# report-kit

ローカルドキュメントの調査タスクを管理するための kanban ワークフロープラグインです。

## 概要

`report-kit` は `/add-report` と `/report` スキルを提供し、ローカルのドキュメント群を体系的に調査・記録するためのワークフローをサポートします。

- 調査タスクを `reports/` ディレクトリで管理
- 詳細なログを `reports/{xxxx}_{title}/log.md` に記録
- 未完了タスクを自動でリマインド

## スキル

### `/add-report`

調査タスクを新規作成します。

```
/add-report [タイトル]
```

- 「知りたいこと」と「目的」を収集してタスクファイルを生成
- `reports/{xxxx}_{title}/{xxxx}_{title}.md` として保存
- 作成後に `/report` を起動するか確認

### `/report`

調査タスクを実行します。

```
/report [タスク番号またはファイル名]
```

- `reports/` から対象タスクを読み込んでドキュメントを網羅的に調査
- 調査結果を `reports/{xxxx}_{title}/log.md` に詳細記録
- 完了時に report ファイルへ `## 調査サマリー` と `## 完了サマリー` を追記

## ディレクトリ構成

プラグインを導入したプロジェクトでは以下のディレクトリが作成されます。

```
reports/
└── {xxxx}_{title}/
    ├── {xxxx}_{title}.md   # タスクファイル（知りたいこと・目的・完了サマリー）
    └── log.md              # 調査ログ（詳細な調査内容）
```

## フック

`check-report.sh` が `UserPromptSubmit` イベントで自動実行され、`reports/` に未完了タスクがある場合にリマインドメッセージを表示します。
