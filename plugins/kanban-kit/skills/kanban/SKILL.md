kanban タスクを実行します。まずプランモードで計画を立て、承認後に実装に移ります。

## 引数

skill tool の args パラメータに値が渡された場合、その値をタスク番号またはファイル名として解釈する（例: `0001` または `0001_add-feature`）。
args が空・未指定の場合は、`kanban/{xxxx}_{title}/{xxxx}_{title}.md` のうち `## 完了サマリー` を含まないものから番号が最大のタスクを自動選択する。

## フェーズ1: プランニング（まず EnterPlanMode でプランモードに入ること）

EnterPlanMode を使ってプランモードに入り、以下の手順で計画を立てること:

1. `kanban/` から対象タスクファイルを読み込み、タスク内容を理解する
2. タスクファイルに「目的」（Why: なぜこの作業が必要か）に該当するセクションがあるか確認する
   - `## 要望` は What/How に該当するため、目的（Why）とはみなさない
   - セクション名は問わない（`## 目的`、`## 背景` など）。内容を読み、作業の動機・背景・理由が記載されているか判断する
   - 目的に該当するセクションが**ない場合**: ユーザーに「目的（Why）に該当するセクションが見つかりません。kanban ファイルに目的セクションを追加してください。」と報告し、プランモードに入らず終了する
3. コードベースを調査し、実装方針を検討する
4. 計画をまとめ、kanban ファイルの `## プラン` セクションに記載する
5. ExitPlanMode でユーザーの承認を待つ
6. ユーザーがリジェクトまたは修正を求めた場合:
   - リジェクトされたプランの内容とユーザーのフィードバックを記憶しておく
   - プランを修正し、kanban ファイルの `## プラン` セクションを更新する
   - 再度 ExitPlanMode で承認を待つ
   - このサイクルを承認されるまで繰り返す

## フェーズ2: 実装（プラン承認後）

> **ログ記録の原則**: ログはインタラクティブセッションで得た情報・判断・経緯の**完全な記録**である。kanban ファイルの `## プラン` は要約版、ログは完全版。情報の要約・省略・圧縮をしてはならない。

承認後は以下の手順で実装を進めること:

1. **最初に**ログファイルを `kanban/{xxxx}_{title}/log.md` に作成し、フェーズ1の成果を記入する
   - ヘッダー、開始時刻（`TZ=Asia/Tokyo date +"%Y-%m-%dT%H:%M:%S+09:00"` で取得）を書く
   - 「タスク概要」に kanban ファイルの要望を転記する
   - 「調査結果」にフェーズ1の調査内容を**省略せず詳細に**記述する
     - 調べたファイルごとに、そのファイルで発見した具体的な事実・構造・パターンを記述する
     - 「〜を確認した」のような結論だけでなく、確認した内容そのものを書く
     - インタラクティブセッションで表示された調査内容と同等の情報量を記録する（要約禁止）
   - 「実装プラン」にインタラクティブセッションで議論した**完全な**実装プランを記述する
     - kanban ファイルの `## プラン` は要約版であり、ログには完全版を書く
     - 調査で発見した具体的なコードパス、検討した代替案とその却下理由、採用アプローチとその理由を含める
     - プランモードでユーザーに提示した内容をそのまま記録する（圧縮しない）
   - 「プランニング経緯」にプランの変遷を記録する
     - 最初に提示したプランの概要
     - ユーザーのフィードバック・リジェクト内容（リジェクトがなかった場合は「初回提案がそのまま承認された」と記載）
     - リジェクトがあった場合は最終プランへの変更内容も記載
   - 「会話内容」にフェーズ1でのやり取りを時系列で記述する
     - ユーザーの指示内容、Claude の提案内容、フィードバック・リジェクトのやり取りを書く
     - 省略せず記述する（「フェーズ1完了」のような要約は不可）
   - 実装フェーズのセクション（編集したファイル、実行したコマンド、判断・意思決定、エラー・問題）は空テンプレートとして書く
2. プランに従い作業を実施する
3. 各ステップ完了時にログファイルへ追記する（編集ファイル・コマンド・判断を記録）
4. 作業完了時:
   - ログファイルの完了日時を更新し最終化する
   - kanban ファイルへ `## 完了サマリー` を追記する（完了日時は JST ISO 8601 形式）

詳細なテンプレートは `references/kanban-workflow.md` を参照すること。

## 注意事項

- **git commit や `/commit` コマンドを自動実行しないこと。** コミットはユーザーが明示的に指示した場合のみ行う。
